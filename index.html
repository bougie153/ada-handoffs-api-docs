  • Endpoints, request/response schemas, and webhook payloads may change before GA
  • Breaking changes will be communicated in advance when possible

Ada Handoffs API

Integrate with Ada's conversation handoff system. These endpoints and webhooks let you send and receive messages during a handoff, manage conversation metadata, transfer files, and receive real-time notifications about handoff lifecycle events.

Endpoints

All endpoints require Bearer token authentication and use the base URL https://{handle}.ada.support/api/v2

Method Endpoint Description
POST /v2/conversations/{id}/messages/ Send a text or file message in a conversation on behalf of an end user, human agent, or system
GET /v2/conversations/{id}/messages/ Fetch paginated message history for a conversation
PATCH /v2/conversations/{id}/ Update conversation metadata (additive merge; set a key to null to remove it)
POST /v2/conversations/{id}/attachments/ Upload a file attachment (max 50 MB) and receive a presigned URL valid for 7 days
POST /v2/conversations/{id}/end-handoff/ End the active handoff and return control to the Ada AI Agent

Webhooks

Ada sends webhook events to your configured endpoint as HTTP POST requests with a JSON payload. Return a 200 status to acknowledge receipt.

Event Description
EVENT v1.conversation.message Fired when a message is sent in a conversation. Includes a handoff_integration field to identify which integration owns the active handoff.
EVENT v1.conversation.handoff.ended Fired when a handoff ends. Signals that control has returned to the AI Agent. Also includes the handoff_integration field.
